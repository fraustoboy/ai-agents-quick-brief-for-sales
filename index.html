Подбор обучения по ИИ-агентам

Шпаргалка для менеджеров Академии по выставлению предложения по запросу "ИИ-агенты"

Шаг 1. Быстрые вопросы клиенту

Задай несколько вопросов, чтобы понять, на какой уровень обучения ориентироваться:

  • Кто будет учиться? Широкая аудитория (менеджеры, аналитики), продвинутые бизнес-пользователи или разработчики / ИТ-команда?
  • Нужен результат без кода или с кодом? Клиент хочет научить людей пользоваться ИИ, собирать процессы в no/low-code или развивать полноценную разработку?
  • Какой тип задачи? Личная продуктивность, автоматизация процессов отдела или создание кастомных ИИ-сервисов в продуктах компании?
  • Насколько быстро нужен результат? Нужно «вчера» на простых инструментах или готовы инвестировать время в команду разработки и архитектуру решений?

Краткая логика выбора:

Нет техбэкграунда → Уровень 1. Системный промптинг
Понимают процессы, но не пишут код → Уровень 2. No/Low-code
Есть разработчики → Уровень 3. Разработка

Шаг 2. Матрица форматов обучения

Сопоставь ответы клиента с матрицей ниже и выбери подходящий уровень. Это шаблон: позже сюда можно будет добавить конкретные курсы.

Уровень Кому подойдёт Требования к знаниям Что предлагаем Результат для клиента Действие
Уровень 1
Системный промптинг (без кода)

Широкая аудитория без технического бэкграунда: менеджеры, аналитики, специалисты любых отделов. Люди, которые хотят использовать ИИ как личного ассистента в повседневной работе.

Базовая компьютерная грамотность и умение формулировать задачи на естественном языке. Готовность пробовать новые инструменты, но без программирования.

Обучение по системному промптингу и работе с ИИ-ассистентами: как ставить задачи модели, задавать роль, стиль, ограничения. Шаблоны промптов под типовые задачи.

Сотрудники понимают, для каких задач полезен ИИ, и умеют самостоятельно использовать его в работе: готовые ассистенты для писем, документов, сводок, идей.

Уровень 2
No / Low-code автоматизация

Продвинутые бизнес-пользователи, аналитики, специалисты по процессам, digital-команды, а также ИТ-специалисты начального уровня, которым нужно быстро автоматизировать рутину.

Понимание бизнес-процессов и базовое алгоритмическое мышление: разложить задачу на шаги, «если / то». Программирование не обязательно, но важна готовность разбираться с настройками систем.

Практикумы по автоматизации процессов и созданию ИИ-агентов на no/low-code платформах (например, n8n и аналоги): как собирать сценарии, подключать сервисы и ИИ-модель.

Участники уходят с первыми рабочими сценариями по своим процессам: заявки, документы, уведомления, отчёты. Появляются агенты, которыми отдел может пользоваться сразу после обучения.

Уровень 3
Полноценная разработка

Команды разработки, ML-инженеры, архитекторы, технические специалисты, которые отвечают за создание и внедрение ИИ-сервисов внутри компании.

Уверенное владение языком программирования (например, Python), опыт работы с API и веб-сервисами. Для продвинутых форматов — знание контейнеризации, баз данных, основ ML.

Инженерные курсы и воркшопы по созданию ИИ-агентов и интеграции ИИ в продукты: разработка логики агента, работа с данными, развёртывание в инфраструктуре заказчика, вопросы качества и безопасности.

Команда умеет проектировать, разрабатывать и поддерживать кастомные ИИ-агенты под задачи бизнеса: глубокая интеграция в существующие системы, контроль качества и стоимости решений.

Эта таблица — базовый прототип. По мере расширения портфеля здесь можно будет расставить конкретные курсы и программы, привязать ссылки на описания и добавить фильтры под отрасль или роль слушателей.
